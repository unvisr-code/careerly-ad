# PRD · Careerly 광고상품 안내 페이지 (Advertise)

> 문서 목적: Careerly에 광고 상품을 판매/운영하기 위한 **광고상품 안내 랜딩 페이지**를 제작하기 위한 요구사항 정의서(PRD)

* 작성일: 2026-01-29
* 대상: careerly.co.kr (careerly-v2)
* 구현 방식: Claude Code로 개발(Next.js 기반 가정)

---

## 1. 배경 및 문제 정의

Careerly는 커뮤니티 피드에서 노출되는 광고 시스템(`banner`, `promoted_post`)을 보유하고 있으나,
광고주/파트너가 상품을 이해하고 문의/집행까지 이어지는 **공식 안내 페이지(Advertise)**가 부재합니다.

현재 광고 문의는 비정형 커뮤니케이션으로 진행되기 쉬워,

* 상품 이해 부족(variant/지면/정책)
* 소재 제출 누락
* 심사 기준 불명확
* 운영 리드타임 증가

를 유발합니다.

**목표:** 광고주가 페이지에서 상품을 이해하고, 필요한 정보를 제출하여 문의/견적 요청까지 자연스럽게 전환되도록 안내 페이지를 구축합니다.

---

## 2. 목표(Goals) & KPI

### 2.1 Goals

1. 광고 상품 구조를 2분 내 이해 가능하게 제공
2. 문의/견적 요청을 표준화된 폼으로 수집
3. 운영 정책/제한 사항을 명확히 고지(분쟁/오해 감소)
4. 내부 운영자가 바로 집행 가능한 수준으로 자료 수집(링크/소재/기간/목표 등)

### 2.2 KPI (런칭 후 4주)

* 페이지 방문 → 문의 전환율(Lead CVR): 목표값은 런칭 후 베이스라인 측정
* 문의 폼 제출 완료율(Completion Rate)
* 부적합 문의 비율(정책 위반/정보 누락)
* 평균 견적/집행 리드타임 감소

---

## 3. 타겟 사용자

1. **광고주(마케터/브랜드)**: 캠페인/프로모션/브랜딩/서비스 소개
2. **채용 담당자/HR**: 채용 홍보, 공고 유입
3. **교육/커뮤니티/이벤트 운영자**: 웨비나/세미나/교육 과정 모집
4. **파트너사(대행사)**: 소재/캠페인 운영 대행

---

## 4. 범위(Scope)

### 4.1 In Scope (MVP)

* 광고상품 소개 랜딩(1페이지 또는 섹션형)
* 상품 라인업(배너 4종 + 프로모션 게시글) 설명
* 노출/운영 방식(placement, position_start/end, Sponsored 표기)
* 소재 가이드(텍스트/이미지/랜딩/UTM)
* 정책 요약(금지/제한 업종, 표기, 랜딩 제한)
* 문의/견적 요청 폼(필수 입력 표준화)
* PDF 다운로드(1~2p 소개서) 링크 제공

### 4.2 Out of Scope (후속)

* 셀프서브형 광고 구매/결제(CPM/CPC)
* 소재 업로드/버전 관리 CMS
* 실시간 리포팅 대시보드
* 광고 인벤토리 예약 캘린더

---

## 5. 브랜드/디자인 가이드(로고 참고 반영)

* 메인 톤: **다크(블랙) 기반**
* 포인트 컬러: 로고 도트 기준 **Careerly Accent = #ED6653** (approx)
* 로고 사용:

  * 심볼(도트+바) 및 워드마크(도트 포함)를 상단/푸터에 배치
  * 배경이 어두운 경우 로고 대비 유지
* UI 스타일:

  * 섹션형 스크롤 랜딩
  * 카드 기반 상품 요약 + 상세 토글/모달
  * CTA(문의하기/자료 다운로드)는 포인트 컬러로 강조

## 6. 정보 구조(IA) & 섹션 구성

### 6.1 URL

* `/advertise` (권장) 또는 `/ads` (대안)

### 6.2 섹션(권장 순서)

1. Hero

   * 헤드라인: “Careerly에서 당신의 캠페인을 커뮤니티에 자연스럽게 노출하세요.”
   * 서브카피: “피드 배너 & 네이티브 프로모션 게시글 제공 · Sponsored 표기 준수”
   * CTA 2개: [상품 다운로드(PDF)] [견적 문의]

2. Why Careerly (간단)

   * 커뮤니티 피드 기반 / 자연스러운 삽입 / 명확한 광고 표기 / 모바일·데스크톱

3. 상품 라인업

   * A. Banner Ads: strip/image/card/native
   * B. Promoted Post

4. 노출/운영 방식

   * placement(현재: feed)
   * position_start/end & display_position(랜덤)
   * 클릭: 새 탭

5. 소재 가이드

   * 타이틀/설명/CTA 권장 글자 수
   * 랜딩/UTM/개인정보 처리방침

6. 정책 요약(심사/제한)

   * 금지 업종/콘텐츠
   * 표기( Sponsored 유지 )
   * 랜딩 제한

7. 문의/견적 폼

   * 광고주 정보 + 캠페인 정보 + 소재/링크 + 목표

8. FAQ

   * 집행 시작 리드타임, 소재 변경, 리포팅, 환불 원칙 등

9. Footer

   * 문의 채널/담당자(내부 입력), 회사 정보, 정책 링크

---

## 7. 핵심 사용자 시나리오(User Stories)

1. 광고주로서, 페이지에서 **배너/프로모션 게시글 차이**를 빠르게 이해하고 싶다.
2. 대행사로서, **제출해야 할 소재 항목**을 명확히 확인하고 싶다.
3. HR 담당자로서, **채용 홍보에 적합한 상품**을 추천받고 싶다.
4. 광고주로서, 정책 위반 없이 집행되도록 **금지/제한 사항**을 확인하고 싶다.
5. 광고주로서, 바로 **견적 문의**를 남기고 담당자에게 회신받고 싶다.

## 8. 기능 요구사항(Functional Requirements)

### 8.1 상품 카드/상세

* 각 상품(variant 포함)은 카드 형태로 요약 노출
* 카드 클릭 시 상세 정보 펼침(Accordion) 또는 모달
* 상세에는 아래를 포함

  * 설명/권장 목적
  * 제출 항목(필드)
  * 예시 카피(짧게)

### 8.2 PDF 다운로드

* 상단 Hero 및 상품 섹션에 PDF 다운로드 버튼
* 파일: `Careerly_광고상품_소개서_1-2p.pdf`

### 8.3 문의/견적 폼

* 폼은 **페이지 내 섹션**으로 제공(또는 별도 `/advertise/contact`)
* 제출 성공 시:

  * 토스트/완료 화면
  * 제출 내용 요약 표시

#### 폼 필수 필드(MVP)

* 회사/브랜드명
* 담당자명
* 이메일
* 연락처(선택 가능)
* 광고 목적(브랜딩/채용/이벤트/전환 등)
* 희망 상품: banner(variant 선택) / promoted_post
* 집행 기간(시작/종료)
* 랜딩 URL
* 소재 링크(이미지 URL 또는 파일 링크, 없으면 “추후 전달” 체크)
* 예산 범위(선택: 범위형)
* 추가 요청사항

#### 폼 검증

* 이메일 형식 검증
* URL 형식 검증
* 필수값 누락 시 인라인 에러

#### 제출 처리(백엔드)

* 옵션 A: 내부 API `POST /api/v1/ads/leads` 신규 구현
* 옵션 B: Slack Webhook/Notion/CRM 연동(서버 액션)

> MVP는 “리드 수집”이 핵심이므로, 기술 난이도 낮은 방식부터 선택(예: Slack + DB 저장)

### 8.4 FAQ

* Accordion 형태
* 최소 6개 문항 제공

## 9. 비기능 요구사항(Non-Functional)

* **SEO**: index 가능한 정적 페이지(SSR/SSG 권장)
* **퍼포먼스**: 이미지 최적화(next/image), LCP 개선
* **접근성**: 명확한 대비, 키보드 탭 이동, aria-label
* **보안**: 폼 스팸 방지(간단 reCAPTCHA/허니팟/레이트리밋)
* **로깅**: 문의 제출 성공/실패 서버 로깅

## 13. 기술 제안(Claude Code 구현 관점)

* Next.js App Router 기준
* `/advertise/page.tsx` + 섹션 컴포넌트 분리
* 폼:

  * 클라이언트 컴포넌트 + 서버 액션(권장) 또는 API route
  * 제출 시 DB 저장(리드 테이블) + Slack 알림
* PDF 다운로드:

  * public 파일로 서빙 또는 S3/스토리지
* 스타일:

  * 기존 디자인 시스템/컴포넌트 활용(버튼/카드/아코디언)

## 14. 수용 기준(Acceptance Criteria)

* [ ] `/advertise` 페이지가 데스크톱/모바일에서 깨짐 없이 렌더링
* [ ] 상품 2종(배너/프로모션) + 배너 4variant 정보가 노출
* [ ] Sponsored 표기/정책 요약 섹션 존재
* [ ] PDF 다운로드가 정상 동작
* [ ] 문의 폼 필수 검증 동작 및 제출 성공/실패 처리
* [ ] 제출 시 내부 채널로 알림(또는 DB 적재) 확인
* [ ] GA4 이벤트 최소 5개 이상 연결

## 16. 부록: 시스템 연동 맥락(참고)

* 광고 타입: `banner`, `promoted_post`
* 배너 variant: `strip`, `image`, `card`, `native`
* 노출: `position_start/end` 범위 내 서버가 `display_position` 계산하여 삽입
* 렌더링:

  * banner → `AdBanner`
  * promoted_post → `CommunityFeedCard (feedType="Sponsored")`
